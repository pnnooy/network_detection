# 融合特征匹配与异常行为分析的网络攻击检测系统 — 选题报告

## 一、项目背景

随着网络攻击手段日益多样化，传统基于单包特征匹配的检测方案面临碎片化告警、误报率高、难以发现未知攻击等挑战。本课题拟构建一个以攻击行为检测为目标的网络入侵检测系统，融合误用检测与异常行为分析双引擎——特征引擎识别已知攻击行为并做同源聚合消除碎片告警，异常引擎基于流量基线发现未知偏离行为，两引擎在行为层面互补，为小型网络环境提供轻量级安全监控能力。

## 二、开发目标

本项目旨在实现一个具备实时抓包分析能力的网络攻击检测原型系统，具体目标包括：

1. 实现基于scapy的网络数据包实时捕获与TCP流重组，解析IP/TCP/UDP/应用层字段
2. 构建可维护的攻击特征库，支持字面匹配与正则匹配，覆盖SQL注入、XSS、木马通信、恶意命令等常见Web攻击；对同源同类特征命中做时间窗口聚合，输出攻击行为告警而非逐包告警
3. 实现基于时间窗口的暴力破解行为检测，监控SSH、FTP、Web登录等敏感端口
4. 建立主机行为基线（并发连接数、访问频次、端口分布、会话时长），识别端口扫描、异常外联陌生IP、内网横向扩散等偏离基线的攻击行为
5. 开发攻击行为监控界面，以攻击行为为粒度汇总展示告警（按行为类型/严重程度/攻击源筛选），支持特征库增删改查，并通过行为关联（behavior_id）将相关告警归入同一攻击事件

## 三、技术路线

系统采用模块化架构，数据流为：**抓包 → TCP重组/协议解析 → 并行检测（特征行为检测 + 统计行为检测）→ 行为告警关联与汇总 → 行为监控GUI**。特征行为检测模块采用多模式匹配算法（AC自动机/KMP），先以暴力子串匹配为baseline，并对同源同类命中做时间窗口聚合；统计行为检测基于滑动窗口与基线阈值法，阈值存放于独立配置文件便于调参。各模块通过统一JSON接口松耦合通信，告警汇总层通过行为关联算法（同源+同类+时间相近）将告警归入攻击行为事件，消除告警碎片化。开发语言为Python（≥3.9），GUI采用tkinter标准库，版本控制使用GitHub Flow分支管理，每人独立feature分支开发，通过Pull Request合并。

## 四、预期成果

交付一套可独立运行的网络攻击检测系统，包含完整源码、攻击特征库、基线配置文件、mock测试数据集、单元测试及最终答辩报告。系统需能够在模拟环境中正确检出预设的各类攻击行为，并通过GUI直观展示告警信息。
